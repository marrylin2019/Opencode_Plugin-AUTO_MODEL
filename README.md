# Opencode_Plugin-AUTO_MODEL

A third-party plugin developed for [Opencode](https://github.com/opencode-ai/opencode) to automatically fetch and inject model lists.

This plugin allows you to configure custom service providers (such as various OpenAI-compatible proxies, relay APIs, local Ollama, etc.) **without manually entering lengthy model lists**. The plugin automatically calls the API in the background to fetch models, and uses heuristic rules alongside `models.dev` data to intelligently populate context lengths, multimodal support (vision, documents), and reasoning model tags.

## Features

- **Auto Discovery**: Supports all standard OpenAI-compatible `/v1/models` endpoints.
- **Seamless Authentication**: Automatically reads Opencode's local `auth.json`, eliminating the need to expose API Keys in plain text within your configuration.
- **Silent Execution**: Runs purely in the background without polluting the Opencode TUI terminal interface.
- **Detailed Logging**: Execution status and error information are automatically archived to a local log file, making it easy to troubleshoot network issues.

## Installation

1. Go to the [Releases page](../../releases/latest) of this repository to download the latest compiled `auto-models.js` file.
2. Save the file to your local directory, for example:
   - Windows: `C:\Users\YourName\.config\opencode\plugins\auto-models.js`
   - WSL/Linux: `/home/YourName\.config\opencode\plugins\auto-models.js`

3. Modify/Add Configuration

Open your Opencode configuration file (usually located at `~/.config/opencode/opencode.jsonc`. If it doesn't exist, please create one first), and import this plugin within the `plugin` array:

```jsonc
{
  "plugin": [
    [
      // The local absolute path to the plugin (For Windows paths, be sure to use file:/// and forward slashes)
      "file:///C:/Users/YourName/.config/opencode/plugins/auto-models.js",
      {
        "providers": [
          {
            "id": "my-proxy",
            "name": "My API Proxy",
            "baseURL": "[https://api.your-proxy.com/v1](https://api.your-proxy.com/v1)"
            // "apiKey": "sk-xxx" // Optional: If the key for this id is already configured in auth.json, this can be omitted.
          },
          {
            "id": "local-ollama",
            "name": "Ollama Local",
            "baseURL": "[http://127.0.0.1:11434/v1](http://127.0.0.1:11434/v1)"
          }
        ]
      }
    ]
  ]
}
```

4. Configure API Key
Use the `/connect` command in opencode-tui -> Select Other Custom Provider -> For Provider id, use the exact same id configured in opencode.jsonc -> Enter your API Key.

Alternatively, you can directly edit `~/.local/share/opencode/auth.json` to add your API Key.

## Local Development and Build
Ensure your system has Bun installed.

```bash
# Clone the repository
git clone [https://github.com/yourusername/opencode-auto-models.git](https://github.com/yourusername/opencode-auto-models.git)
cd opencode-auto-models

# Install dependencies
bun install

# Build the plugin
bun build src/index.ts --outfile=auto-models.js --target node --external @opencode-ai/plugin
```

## Troubleshooting / Logs
If models are not loading properly, you can check the log file generated by the plugin:

Path: `~/.config/opencode/opencode-auto-models.log`
